安装搭建hexo

首先需要安装Node.js

安装好之后，在终端中输入node -v 和npm -v 查看node.js 和npm的版本

~ % node -v
v14.16.1
~ % npm -v
6.14.12

利用npm包管理器来安装hexo框架，由于国内的镜像源速度很慢，先利用npm安装一个cnpm 指向淘宝的镜像源

npm install -g cnpm --registry=https://registry.npm.taobao.org

利用cnpm全局安装hexo博客框架

cnpm install -g hexo-cli

输入hexo -v 判断hexo是否安装成功

~ % hexo -v

开始搭建hexo博客

在本地新建一个文件夹，用来存放博客的相关文件（博客有问题了直接删除重来，记得时不时备份一下）

通过终端，到Blog文件下，进行初始化，将该文件夹初始化为博客文件夹

sudo hexo init

初始化成功后，Blog文件夹内会生成一系列的文件

初始化ok后，我们输入hexo的相关命令，进行启动

hexo s

到浏览器中去输入http://localhost:4000 访问博客，这个时候的博客是默认的主题，并且默认给你生成了一篇hello-world的博客。这样你的hexo博客框架本地就搭建好了。

hexo具体相关的命令和配置等操作，都可以在hexo官方文档中查看

Hexo主题设置

下载主题文件

打开hexo根目录下的_config.yml文件(修改此处指定的主题文件名为你刚刚复制粘贴进去的文件名)

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: hexo-theme

期间遇到过一些问题，一一列举一下

hexo d 报错

#执行部署网站命令
hexo d

报错：

error：spawn failed…

fatal: cannot lock ref ‘HEAD’: unable to resolve reference HEAD: Invalid argument error: src refspec

error: src refspec HEAD does not match any.等等

百度了一些文章，发现大多是因为git进行push或者hexo d 的时候改变了一些.deploy_git文件下的内容

解决方法：

#删除.deploy_git文件夹
sudo rm -rf .deploy_git/
#执行该命令
git config –global core.autocrlf false

然后再依次执行hexo的命令即可

#清除缓存文件 (db.json) 和已生成的静态文件 (public)。

hexo clean

#生成静态文件。

hexo g

#部署网站

hexo d

部署GitHub pages 自定义域名失效的问题

首先需要解析域名，我是腾讯云申请的域名，以此为例（其他平台的可以对应的百度搜索一下）

解析

等待审核通过之后，将自己的二级域名设置到GitHub pages中

发现设置的确成功了，但是每次hexo d 之后，设置的自定义域名又不在了，导致每次使用自定义域名访问博客报错404

所以需要在hexo根目录下source中添加一个CNAME文件，内容为你的域名（不包括www.或http等等）

注意点：

命名为 CNAME，全部大写，没有后缀。
内容只包含一个域名，且不含 www. 或 http 等。
放到hexo根目录下的source文件夹中。

将文件放入hexo根目录下的source文件夹中后，执行hexo d，再去GitHub settings 查看GitHub pages 是否更新成功。

在GitHub pages设置自定义域名后，只用域名访问显示异常

设置好域名后，请求访问，发现博客的主题样式都没有了

研究咨询后才知道还需要在hexo根目录下的_config.yml 中进行配置，将URL修改为自己的自定义域名即可

搭建hexo的时候，执行hexo d 报错：ERROR Deployer not found: git

npm install --save hexo-deployer-git

还有在下载插件的过程中遇到的一个报错问题

xcode-select: error :command line tools are already installed, use “Software Update” to install updates

Remove the old tool in your terminal

sudo rm -rf /Library/Developer/CommandLineTools

Install it again

xcode-select --install
